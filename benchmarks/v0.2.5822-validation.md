## Rosetta / x86_64 macOS

Issue #504 was validated locally under Rosetta.

| Binary | Signature | Command | Result |
| --- | --- | --- | --- |
| Published `v0.2.5821` `codedb-darwin-x86_64` | Developer ID signed | `arch -x86_64 ./codedb-darwin-x86_64 --help` | exit `139` |
| Fresh `0.2.5822` `x86_64-macos` build | unsigned | `arch -x86_64 zig-out/bin/codedb --help` | exit `0` |
| Fresh `0.2.5822` `x86_64-macos` build | unsigned | `arch -x86_64 zig-out/bin/codedb --version` | exit `0`, `codedb 0.2.5822` |
| Same `0.2.5822` x86_64 binary, manually ad-hoc signed | ad-hoc signed | `arch -x86_64 ./patched-x86-manual-signed --help` | exit `139` |
| Fresh `0.2.5822` `aarch64-macos` build | ad-hoc signed | `zig-out/bin/codedb --help` | exit `0` |

The release fix is therefore to avoid codesigning the Intel macOS release
artifact until the upstream Zig 0.16 / Mach-O signing crash is resolved. The
arm64 macOS release path still signs when a signing identity is configured.

## SWE-bench Lite smoke

Four subagents exercised `codedb 0.2.5822` on real SWE-bench Lite instances.
Each worker used a non-temporary workspace because codedb intentionally refuses
to index `/tmp` roots.

| Instance | Repo | codedb navigation | Test result | Notes |
| --- | --- | --- | --- | --- |
| `pallets__flask-4992` | `pallets/flask` | `search "from_file"`, `outline src/flask/config.py`, `find Config`, post-patch `search "mode=\"rb\""` | `1 passed in 0.02s` | Added a `mode` parameter to `Config.from_file` and TOML regression coverage. |
| `pytest-dev__pytest-5221` | `pytest-dev/pytest` | `search "def _showfixtures_main"`, `outline src/_pytest/python.py`, `find _showfixtures_main`, `read -L 1303-1357 src/_pytest/python.py` | `2 passed in 0.06s` | Required `PYTEST_DISABLE_PLUGIN_AUTOLOAD=1` for the old pytest checkout. |
| `sympy__sympy-12454` | `sympy/sympy` | `search "def is_upper"`, `outline sympy/matrices/matrices.py`, `find is_upper`, `read -L 1100-1117 sympy/matrices/matrices.py` | `2 passed, 150 deselected`; `2 passed, 50 deselected` | Used Python 3.9 venv because the old SymPy checkout is not compatible with host Python 3.14. |
| `psf__requests-2317` | `psf/requests` | `search "builtin_str(method)"`, `outline requests/sessions.py`, `find Session`, `read requests/sessions.py --compact` | Target pytest blocked | Old Requests collection fails on host Python 3.14 with `ModuleNotFoundError: No module named 'cgi'`; a direct smoke confirmed `'get'` and `b'GET'` normalize to `GET`. |

This is not a full SWE-bench score. It is a release smoke intended to catch
obvious real-world regressions in indexing, search, outline, read/symbol lookup,
and edit-support workflows across several Python codebases.
